Getting Started with DocsBot

This is a sample note demonstrating how DocsBot renders project documentation in a modal.

Project Structure

Each project lives in a directory with this layout:

projects/my-project/
  data/
    meta.js       -- project identity and navigation
    research.js   -- research directions (R1, R2, ...)
    backlog.js    -- engineering tasks (P0-01, P1-02, ...)
    roadmap.js    -- weekly planning
    changelog.js  -- commit history
    notes.js      -- note index
  notes/
    my-note.html  -- individual notes (like this one)

Quick Start

  1. Copy the examples/demo/ directory to projects/your-project/.
  2. Edit the data/*.js files to match your project.
  3. Run docsbot serve and open the dashboard.

Data Format

All data files are plain JavaScript that define window.AUGUR_* globals. The frontend parses them in a sandbox and renders cards. No build step is required.
