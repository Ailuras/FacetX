Architecture Overview

This note describes the high-level architecture of a sample project using the DocsBot data model.

Component Map

ComponentLanguagePurpose
ParserC++Read input into AST
EncoderC++Transform AST to solver format
Solver WrapperC++Dispatch to external solver
ValidatorPythonCheck candidate soundness
Batch RunnerPythonOrchestrate experiment rounds

Data Flow

  1. Input → Parser produces AST.
  2. AST → Encoder produces solver-ready formula.
  3. Formula → Solver returns SAT / UNSAT / UNKNOWN.
  4. Candidates → Validator checks entailment before use.
  5. Results → Batch Runner collects manifests.

Research → Engineering Mapping

Each research direction (R) in research.js maps to one or more engineering tasks (P) in backlog.js through the serves and depends_on fields. This cross-reference is what makes the dashboard useful: you can see which engineering work is advancing which research goal.
